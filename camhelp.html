⚙️ Assembly & Deployment Protocol Internal Use Only

Standard Operating Procedure for setting up, cleaning, and provisioning NVIDIA Jetson Orin Nano systems before delivery to the end client.

Phase 1: Factory Reset (Refurbishing a Used Device)
CRITICAL: If this unit was previously used for testing or by another client, you MUST wipe the state files. If you skip this, the new client will be locked out of their own bot.

Open the Jetson terminal via SSH and navigate to the project directory. Execute the following commands sequentially to destroy all previous telemetry, photos, and permissions:

# 1. Stop the running service
sudo systemctl stop visioncore.service

# 2. Delete all evidence photos
rm -rf evidence_hd_blackbox/*.jpg

# 3. Delete all State and Database JSON files
rm access_control.json
rm system_state.json
rm schedule.json
rm masked_zones.json
rm downtime_history.json
rm last_alive.txt

# 4. Clear the system logs
> system.log

Verification: The access_control.json file must not exist. When the script boots again, it will automatically recreate a blank one, ready for the new owner to claim.

Phase 2: Provisioning the Telegram Bot

Each client needs a unique Bot Token. You must create this via Telegram before editing the code.

  • 1 Open the Telegram App and search for the user @BotFather.
  • 2 Send the command /newbot.
  • 3 Name the bot (e.g., Smith Residence Security).
  • 4 Choose a username ending in "bot" (e.g., smith_guardian_bot).
  • 5 Copy the HTTP API Token provided by BotFather. It looks like this: 1234567890:ABCdefGhIJKlmNoPQRstuVWXyz.
UX Setup: Send the command /setuserpic to BotFather and upload the client's logo or a generic camera icon to make the bot look professional.
Phase 3: Python Configuration (.py)

Open the main Python script (e.g., vision_core.py) using nano or VS Code.

Locate the class Config: section at the top of the file. You must modify the following variables for the specific client:

class Config:
    # --- 1. TELEGRAM CONFIGURATION ---
    # Paste the token generated by BotFather here:
    TELEGRAM_TOKEN   = "YOUR_NEW_TOKEN_HERE"
    
    # Leave this ID as is, or put a random string. 
    # It is only used for the boot message. The client will take over dynamically.
    TELEGRAM_CHAT_ID = "8574599572" 

    # --- 2. CAMERA CONFIGURATION ---
    # Set the client's local static IP and RTSP credentials
    CAM_IP   = "192.168.1.XX"
    CAM_USER = "admin"
    CAM_PASS = "FactoryPassword123"

Important Notes on IP Subnets:

Ensure the Jetson Orin Nano and the PoE Camera are configured on the same subnet as the client's router (e.g., 192.168.1.X or 10.1.10.X). Set a Static IP for the Jetson in the Ubuntu Network Settings.

Phase 4: Web Panel HTML Deployment

If you are deploying the Web App control panel for the client:

  • 1 Upload the index.html file to the client's web server or local Nginx/Apache instance on the Jetson.
  • 2 Open the HTML file and verify that the <title> tag reflects the client's property (e.g., <title>Smith Residence - Control Panel</title>).
Phase 5: Final Testing & Client Handover

Once the hardware is installed on-site, follow this strict procedure to hand over the digital keys securely:

  • 1 Reboot the Jetson device to ensure the visioncore.service auto-starts cleanly.
  • 2 Ask the Primary Client (The Owner) to open Telegram on their phone and search for the bot's username (e.g., @smith_guardian_bot).
  • 3 Instruct them to press START (or send /start).
SUCCESS METRIC: The client should instantly receive a message stating:
"👑 INITIAL CONFIGURATION: You are now the Super Admin."

Because the access_control.json was wiped in Phase 1, the system binds its master ownership to the very first Telegram ID that contacts it.

Adding Family or Staff

Instruct the owner that if they want their spouse or staff to have access, the staff member must find the bot and send a message. The owner will receive an alert with an ✅ APPROVE ACCESS button to authorize them.
