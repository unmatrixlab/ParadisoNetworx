VISION CORE V2

Sysprep & Factory Provisioning Guide (0 km)

1 System Architecture Overview

Unlike older versions, Vision Core V2 uses an Externalized Credentials Architecture with an Auto-Healing Vault. You must NEVER hardcode client passwords directly into the Python source code.

To prepare a newly cloned Jetson drive for a client, you will use the automated Provisioning Script (Sysprep).

2 Run the Provisioning Script

Open the Jetson terminal, navigate to your project folder, and execute the following command:

python3 preparar_equipo.py

The script will ask for 5 essential parameters. If you press ENTER without typing anything, it will use the default value shown in the brackets [ ].

1. Camera IP [192.168.1.17]: 
2. Camera User [admin]: 
3. Camera Password [123456]: 
4. Telegram Bot Token [8299472...]: 
5. Admin Chat ID [8574599572]:

⚠️ What happens in the background?

Once you hit enter on the last question, the script performs a "0 km Factory Reset":

  • Creates credenciales.json with atomic write protection.
  • Creates an untouchable backup vault: credenciales_backup.json.
  • WIPES all old client data: Photos, Downtime Logs, Schedules, and Masked Zones.
  • DELETES access_control.json so the bot forgets previous owners.

3 Auto-Healing Verification

The system is now fully provisioned. Start the main engine as usual:

python3 guardian_produccion.py

Fail-Safe Mechanism: If the Jetson suffers a power outage and the primary credenciales.json gets corrupted, the system will automatically detect the failure, extract the clean data from the Backup Vault, rebuild the missing file, and boot the camera without crashing.

4 Telegram Ownership (Super Admin)

Because the provisioning script wiped the access control database, the bot is currently "unclaimed".

  • Claiming the System: The very first person to open Telegram and send a message (e.g., /start) to the bot will automatically be registered as the Super Admin.
  • Adding Users: The Super Admin must approve all future users. If the client wants to add their family members, those members must message the bot, and the Super Admin will receive an alert with an Approve Access button.
